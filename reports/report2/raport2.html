1. Abstract

This page represents the architecture and the documentation of our web application that is further described in the following sections. It contains the functionalities, the usecases and some diagrams meant to show the guidelines that we want to follow in order to implement our application. The technologies used are presented at the 4th point of this page.

2. Project description

Our application's aim is to create a pacman experience of our faculty (Faculty of Computer Science - Iasi). The opponents are characters created by the computer. The user will be represented based on his Facebook profile image. For every level of the game a new map will be rendered, spanning through a number of floors in the faculty. There will be offered coins and powerups in order to increase the score. In the map, there will be present some "ghosts" that will represent the teachers of the user.

3. Functionalities

User's Usecases

  • when the user enters the application, the login page will appear;
  • after clicking the -login with facebook- button a popup provided with Firebase will appear;
  • the user enters his/her credentials and then manages or not to login;
  • after the login phase, the user is redirected to the Start Game page, where he/she has to click the -Start Game- button in order to continue;
  • after clicking the -Start Game- button, the user is redirected to the first level of the game;
  • if the user wins the current level, he/she will be prompted with a popup from which he/she can choose to go to the next level or to logout from the application;
  • if the user loses the game, he/she will be prompted with a GameOver message and with the possibilites to retry playing or logging out from the application;
  • when the user reaches the final level, he/she will be prompted with a congratulations message and the possibilites to retry the game or to logout from the application;

Single Page Application

This is a Single Page Application, as it is loads a single HTML page and then dynamically updates it as the user interacts with it.

4. Technologies

Microdata

HTML5 Microdata is used by pairing the element tags with a specific HTML attribute meant to embed machine-readable data in our document. The search engines, browsers and web-crawlers will extract the microdata, all to provide a rich browsing experience for the users.

Web Storage

The credentials provided by the user are stored in localStorage, keeping in this way the user's data in the browser in the eventuality that he/she will want to remain logged in, in order to play the game. In this way, we can store more information rather than using cookies, without affecting the application's performance.

HTML, CSS & JS

HTML, CSS and JS are used for implementing the design and the game core of our application.

Firebase authenication

Because our application needs to know the identity of the user to provide a customized experience, Firebase was used.

Node.js

Node.js is used for the server part. The server sends the entire page and after that, it won't need a network connection to make the most of the functionalities provided.
